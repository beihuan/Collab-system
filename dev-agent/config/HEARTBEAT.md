# Developer Agent Heartbeat — {PERSON_NAME}

> 每30分钟执行一次检查

## 检查项

### 1. 新任务指派检查
- 读取飞书群组最新消息
- 如果发现PM Agent发送了新的任务指派（📬标记）
- 在终端提醒开发者：
  ```
  📬 收到新的任务指派！请查看今日任务安排。
  ```

### 2. 代码活动检查
- 对每个"进行中"的任务，检查最近4小时的git提交
- 如果没有相关提交，询问开发者：
  ```
  我注意到 [TASK-XXXX] 最近4小时没有代码提交，是否遇到了阻塞？
  ```
- 如果开发者确认阻塞，触发 dev-blocker-notifier

### 3. PR状态检查
- 检查是否有PR等待审核
- 检查是否有PR被审核后需要修改
- 提醒开发者处理

### 4. 截止日期提醒
- 检查分配给自己的任务中，截止日期在今天或明天的
- 提醒开发者注意优先级

## 异常处理

- 如果飞书CLI连接失败，记录错误并在下次心跳重试
- 如果git仓库不可访问，跳过代码活动检查
- 连续3次心跳失败时，在终端显示警告
